## Setting Up Webhooks

Webhooks enable real-time updates from Cercle Talents to your systems. Configure them in your dashboard under Settings > Webhooks.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to the Webhooks section and click "New Webhook".
    
    Specify the event type, such as `candidate.updated` or `job.applied`.
  </Step>
  <Step title="Add Endpoint URL" icon="link">
    Enter your secure endpoint, e.g., `https://your-webhook-url.com/webhook`.
    
````javascript
{
  "event": "candidate.updated",
  "data": {
    "id": "cand_123",
    "name": "John Doe",
    "status": "matched"
  }
}
````
  </Step>
  <Step title="Verify and Test" icon="check-circle">
    Send a test event and verify receipt on your server.
    
    Enable the webhook once confirmed.
  </Step>
</Steps>

## API Authentication

Authenticate API requests using Bearer tokens. Generate your token in the dashboard under API Keys.

<Tabs>
  <Tab title="JavaScript" icon="code">
    Use `fetch` with the Authorization header.
    
````javascript
const response = await fetch('https://api.example.com/v1/candidates', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${YOUR_API_KEY}`,
    'Content-Type': 'application/json'
  }
});
````
  </Tab>
  <Tab title="Python" icon="python">
    Use `requests` library for authenticated calls.
    
````python
import requests

headers = {
    'Authorization': f'Bearer {YOUR_API_KEY}',
    'Content-Type': 'application/json'
}
response = requests.get('https://api.example.com/v1/candidates', headers=headers)
````
  </Tab>
</Tabs>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token for authentication: `Bearer YOUR_API_KEY`.
</ParamField>

## API Endpoints

Key endpoints for managing candidates and jobs.

<Request tabs="JavaScript,cURL" show-lines="true">
````javascript
const response = await fetch('https://api.example.com/v1/candidates', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${YOUR_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Jane Smith',
    skills: ['JavaScript', 'Recruitment'],
    experience: 5
  })
});
````
````bash
curl -X POST https://api.example.com/v1/candidates \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Smith",
    "skills": ["JavaScript", "Recruitment"],
    "experience": 5
  }'
````
</Request>

<Response tabs="200,400">
````json
{
  "id": "cand_456",
  "name": "Jane Smith",
  "skills": ["JavaScript", "Recruitment"],
  "status": "active"
}
````
````json
{
  "error": "Invalid token",
  "message": "Authentication failed"
}
````
</Response>

<ResponseField name="id" field-type="string" required="true">
  Unique candidate identifier.
</ResponseField>

<ResponseField name="status" field-type="string">
  Current status: `active`, `matched`, or `rejected`.
</ResponseField>

## Building Custom Integrations

For advanced data sync, build custom scripts to pull candidates into your internal database.

<CodeGroup tabs="Node.js,Python">
````javascript
// Node.js example: Sync candidates daily
const cron = require('node-cron');
cron.schedule('0 0 * * *', async () => {
  const res = await fetch('https://api.example.com/v1/candidates?limit=100', {
    headers: { 'Authorization': `Bearer ${process.env.API_KEY}` }
  });
  const candidates = await res.json();
  // Sync to your DB
  console.log(`Synced ${candidates.length} candidates`);
});
````
````python
# Python example: Sync with cron job
import requests
import schedule
import time

def sync_candidates():
    headers = {'Authorization': f'Bearer {YOUR_API_KEY}'}
    res = requests.get('https://api.example.com/v1/candidates?limit=100', headers=headers)
    candidates = res.json()
    # Sync to your DB
    print(f'Synced {len(candidates)} candidates')

schedule.every().day.at('00:00').do(sync_candidates)
````
</CodeGroup>
